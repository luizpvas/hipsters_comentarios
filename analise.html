Análise de comentários do hipsters.tech¶

Esse um projeto quase pessoal para mim. Eu sou muito f do hipsters, e recomendo fortemente que todos escutem pelo menos um episdio. Um dos meus favoritos sobre Testes A/B.

Vamos comear com uma descrio geral dos dados. Eu implementei um crawler com a biblioteca Puppeteer para extrair algumas informaes sobre os episdios (ttulo, data, participantes e tags) e os comentrios. O Hipsters usa Disqus no sistema de comentrios, e at onde sei eles no oferecem uma API para esse tipo de extro. O cdigo do crawler est no repositrio desse projeto, se voc tiver curiosidade.

comments = []
episodes = []

for page_id in files:
    file = !cat data/{page_id}
    episode = json.loads(file[0])
    episodes.append(episode)
    date_parts = episode['releaseDate'].split('/')
    release_date = date_parts[2] + '-' + date_parts[1] + '-' + date_parts[0]
    for comment in episode['comments']:
        comments.append({
            'titulo': episode['title'],
            'data': release_date,
            'tags': episode['tags'],
            'participantes': episode['people'],
            'autor_comentario': comment['author'],
            'comentario': comment['comment']
        })

df = pd.DataFrame(comments)

1. Quais assuntos geram mais engajamento do público?¶

Engajamento algo muito mais abrangente (e subjetivo) do que a qtde. de comentrios, mas o dado que temos pra hoje. Ns podemos agrupar por tags (cada episdio possui uma ou mais tags) para descobrir os assuntos que gera mais engajamento do pblico.

Na esquerda mostrado uma contagem absoluta, e na direita a mdia por tag. A contagem no legal porque existem, por exemplo, muito mais episdios sobre programao do que startup, fazendo a qtde. de comentrios ser maior. Se a gente tirar a mdia, a qtde. de comentrios normalizada pela qtde. de episdios, o que gera uma mtrica igual para todas as tags.

Web, front-end, mobile e infra so os 4 assuntos mais comentados, e edio de vdeo ficou por ltimo.

1. Quais os episódios mais amados ♥?¶

Anlise de sentimento uma coisa complicada. E abrangante. E vaga. Infelizmente, o Tone Analyzer do Watson s funciona para ingls e francs no momento, ento a gente vai ter que se virar com... contagem de palavras. Essa a abordagem mais simples para anlise de sentimento, mas o que temos para hoje.

Eu vou usar a base OpLexicon V3.0, que contm palavras positivas, negativas e neutras.

positive = []

with open('lexico_v3.0.txt') as f:
    lines = f.read().split('\n')
    for line in lines:
        if ',' in line:
            word, type_, score, _nsei = line.split(',')
            if score == '1':
                positive.append(word)

def build_spacy_doc(comentario):
    return nlp(comentario)

def split_tokens(doc):
    tokens = []
    for t in doc:
        if not t.is_stop and not t.is_punct and t.is_alpha:
            tokens.append(t.lower_)
    return tokens

def atribuir_score_positivo(doc):
    score = 0
    for token in doc:
        if token.lower_ in positive:
            score += 1
    return score

df['doc'] = df['comentario'].apply(lambda x: build_spacy_doc(x))    
df['positivo'] = df['doc'].apply(lambda x: atribuir_score_positivo(x))
df['tokens'] = df['doc'].apply(lambda x: split_tokens(x))

contagem_positivo = df.groupby('titulo').agg({'positivo': 'sum', 'comentario': 'size'})
contagem_positivo.rename(columns={'comentario': 'qtde_comentarios', 'positivo': 'palavras_positivas'}, inplace=True)

Quais os principais temas/palavras-chave por assunto?¶

Para extrair as palavras-chave, ns vamos usar o algoritmo TF-IDF. O TF-IDF puramente "estatstico" - ele apenas se preocupa com qtde. de palavras em cada documento e a frequncia das palavras na coleo. A ideia por trs do TF-IDF que as palavras importantes para um documento vo aparecer vrias vezes nesse documento e poucas vezes nos outros.

Nesse caso, ns vamos rodar o TF-IDF apenas curiosidade, mas ele poderia nos ajudar em outros casos:

  • Filtrar stopwords que no esto presentes na lista padro das bibliotecas (lembre-se, o IDF pode chegar a 0 se a palavra aparecer em todos os documentos da base).
  • Usar como um dos atributos de um modelo de predio para classificar comentrios por assunto.
  • Ajudar na interpretao do resultado modelagem de tpico (como LDA)

from gensim.corpora import Dictionary
from gensim.models import TfidfModel

# O resultado ficou um pouco mais relevante excluindo os episdios com mais comentrios.
top_episodios = df.groupby('titulo').count().reset_index().sort_values('comentario', ascending=False).head()['titulo']

docs_per_tag = {}
for index, row in df[~df['titulo'].isin(top_episodios)].iterrows():
    for tag in row['tags']:
        if tag != 'Podcast':
            if tag in docs_per_tag:
                docs_per_tag[tag] += ' ' + ' '.join(row['tokens'])
            else:
                docs_per_tag[tag] = ' '.join(row['tokens'])

words = [text.split(' ') for text in docs_per_tag.values()]
dct = Dictionary(words)
corpus = [dct.doc2bow(line) for line in words]
model = TfidfModel(corpus)

for tag, text in docs_per_tag.items():
    item = [t for t in dct.doc2bow(text.split(' ')) if t[1] > 2]
    scores = sorted(model[item], key=lambda x: -x[1])
    #print(item)
    palavras = [dct[score[0]] for score in scores[:10]]
    print('Tag: {}'.format(tag))
    print(' '.join(palavras))
    print('')
